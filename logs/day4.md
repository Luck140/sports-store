日志 Day4：2026年6月9日

今日工作概述

  今天的开发工作从上午开始，主要任务是对前端界面进行系统性优化。首先完成了项目运行环境的搭建，启动后端 FastAPI 服务和前端 Vite 开发服务器，确认前后端数据交互正常。随后对现有界面进行了整体评估，发现存在配色缺乏辨识度、表单无前端验证、异步操作无加载状态、页面在手机上布局溢出等问题。为了更科学地进行改进，引入了 Impeccable 开源设计工具对项目进行自动化诊断扫描，结合人工评审制定了完整的改进计划。整天的改进按照五个步骤推进，从建立设计系统文档开始，到替换配色方案、修复响应式布局、增强表单验证和交互反馈，最后完成代码清理和无障碍优化。


一、版本迭代过程

  今天一共经历了三个版本，前两个版本不太理想，第三个版本才确定了正确的实施路径。

  1. 版本一

  在还没有建立设计规范的情况下直接让 AI 修改 Home.vue 文件。AI 生成的新首页使用了深色背景搭配橙红色系的风格，实际效果比较杂乱，各种装饰形状和动画堆砌在一起，不是我想要的感觉。

  2. 版本二

  经过反思，发现问题的根源在于没有先确定设计方向就直接改代码。没有设计规范约束的情况下做出来的东西缺乏整体感。于是完全放弃了第一个版本，将代码恢复到最初的状态。

  3. 版本三

  恢复代码之后，按照审计报告制定的五个维度逐步推进。第一步建立设计文档明确品牌定位和配色方向，第二步建立 CSS 变量体系替换全站硬编码颜色，第三步修复响应式布局，第四步添加表单验证和加载状态，第五步清理代码和添加无障碍支持。每一步都在前一步的基础上进行，保证了修改的条理性和可控性。


二、配色方案说明

  最终选定的配色是深松绿搭配琥珀金，辅以暖白色背景。

  1. 主色选择

  市面上大部分企业管理系统都在使用蓝色作为主色，Element Plus 的默认主题也是蓝色。如果继续使用蓝色，界面会显得很大众化。松绿色来自森林和自然的意象，既有深度和稳重感，又不会显得沉闷，在企业管理系统中比较少见，能够形成独特的品牌记忆点。

  2. 强调色选择

  在深绿色的基础上需要一个能够形成对比的颜色。琥珀金在色环上与绿色形成互补关系，金色本身带有价值感和品质感的联想，用在价格显示和菜单选中态上可以传递正面信号。

  3. 背景色调整

  原始界面使用冷灰色背景，虽然干净但缺乏温度。调整为暖白色之后，配合深绿色的主色，整体的氛围更加协调自然。所有的颜色值都被定义在 base.css 文件的根变量中，Element Plus 的组件主题也通过 CSS 变量进行了覆盖，按钮、标签、表格、弹窗、菜单等组件都能够自动适配新的配色方案。


三、详细改进内容

  1. 设计系统搭建

  在没有设计规范的情况下直接修改代码容易出现风格不统一的问题。建立了两个设计文档。PRODUCT.md 记录了产品层面的战略信息，包括用户画像为体育用品批发行业的管理和工作人员，品牌个性定义为专业可靠高效，设计原则方面明确了拒绝花哨电商风格。DESIGN.md 记录了视觉层面的具体规范，包括颜色的色值用途、字体层级、组件样式定义和具体的使用规范。

  2. CSS Token 系统的建立

  在 base.css 中定义了一套完整的变量体系，涵盖了品牌色、中性色、文字色、暗角色、反馈色、阴影、字体大小、间距和圆角等维度。Element Plus 的组件主题变量与项目变量进行了映射，实现了组件样式的一键适配。后续如果要更换配色，只需要修改变量值即可，不需要到每个页面文件中去查找和替换颜色。

  3. 响应式布局修复

  登录页面和注册页面原本使用了固定的像素宽度，在手机上会出现水平滚动条。将固定宽度改为百分比宽度加最大宽度的组合方式，在不同屏幕尺寸下调整上下边距。首页的功能卡片原本使用固定宽度，改为 clamp 函数让卡片宽度在最小值和最大值之间自适应。商品列表的栅格系统添加了响应式断点参数，手机上每行一列，平板上每行两列，桌面上每行四列。侧边栏在平板尺寸下宽度缩小，在手机尺寸下改为横向布局。

  4. 表单验证与交互反馈

  登录页面添加了用户名和密码的必填验证。注册页面添加了更多的验证规则，密码要求至少六位，手机号需要符合中国大陆的手机号码格式，邮箱需要符合标准格式。购物车页面在提交订单前验证收件人、联系电话和收件地址。忘记密码页面验证用户名和邮箱。加载状态的添加覆盖了七个页面，所有表单的提交按钮绑定了 loading 属性防止重复提交。搜索功能添加了三百毫秒的防抖处理，避免用户快速输入时产生大量无效请求。

  5. 代码清理与无障碍优化

  删除了项目中未使用的计数器功能 Store 文件。修复了路由守卫中已经废弃的 next 函数调用，消除了控制台警告。侧边栏菜单的颜色通过 CSS 变量统一控制，替代了之前在组件属性中硬编码颜色的方式。添加了对 prefers-reduced-motion 媒体查询的支持，在用户开启减少动效选项时禁用所有动画和过渡效果。


四、遇到的困难

  1. Element Plus 组件样式覆盖

  虽然 Element Plus 提供了 CSS 变量机制，但不是所有的组件样式都能通过变量覆盖，有些内部样式使用了固定色值，需要查看源码确认具体的 CSS 选择器，反复调试才能生效。

  2. 配色选择的反复

  一开始保留了蓝色系配色，但用户反馈太大众化，需要完全放弃蓝色系重新选择。深松绿加琥珀金的组合试了几次才确定下来，中间还考虑过其他搭配方案。

  3. Windows 环境兼容性

  部分脚本在路径处理上使用 Linux 风格的分隔符需要调整，杀进程操作也需要使用 Windows 专用的命令格式。
