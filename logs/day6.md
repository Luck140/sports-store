一、全流程功能测试

  今天花了大量时间对系统的完整业务流程进行了逐一测试，模拟真实用户的使用场景来验证各功能模块之间的衔接是否顺畅。

  1. 顾客端流程测试

  从未登录状态开始，访问首页确认热销商品和轮播横幅正常展示。点击商品分类入口确认能正确跳转到对应分类的商品列表。浏览商品列表时逐一测试分类筛选、价格排序、关键词搜索和价格区间过滤功能，确认各个筛选条件能正确组合且分页正常工作。点击商品进入详情页，确认商品信息、库存状态和已有评价正确显示。尝试在未登录状态下点击加入购物车，确认系统正确提示需要登录并跳转到登录页。注册一个新账号后测试登录功能，确认登录后自动跳转到商品列表。登录后再次浏览商品并加入购物车，确认购物车角标数字实时更新。进入购物车页面，确认商品名称、单价、数量和金额都正确显示。修改购买数量后确认小计和总价实时刷新。从已保存的地址中选择一个收货地址确认自动填充。填写货物重量后确认运费正确计算。选择开具发票后确认发票字段正确显示。提交订单后确认跳转到订单详情页。对订单进行支付操作，确认支付弹窗展示完整费用明细。管理员确认订单并发货后，在顾客端确认收货，然后对商品进行评价。

  2. 管理员端流程测试

  使用管理员账号登录后，先检查仪表盘的待办提醒是否正常显示。进入订单管理页面，对刚提交的测试订单进行确认操作，确认库存扣减正确。对已确认的订单进行发货操作，确认运费自动计算。进入进货管理页面，创建一份进货单并确认入库，确认库存数量正确增加。打开报表中心，检查各个报表的数据是否与实际操作记录一致。对顾客提交的退款申请进行审核，先拒绝再同意，确认订单状态和库存都能正确回退。

二、测试中发现的问题与修复

  通过全流程测试，发现了一些细节问题并进行了修复。

  1. 部分页面在不同浏览器下的兼容性问题。在 Chrome 和 Edge 中测试了所有页面，发现 Element Plus 的某些组件在 Edge 的旧版本中渲染效果略有差异，主要是圆角和阴影的显示。通过调整 CSS 变量确保了两个浏览器的显示一致性。

  2. 商品列表在无筛选结果时的空状态提示不够友好。之前只显示了简单的文字提示，现在增加了一个清除筛选条件的按钮，用户可以一键重置所有筛选条件查看全部商品。

  3. 购物车修改数量时没有对库存上限进行检查。如果用户尝试购买超过库存数量的商品，应该给出提示并限制最大购买数量。修复后数量输入框的最大值会自动绑定到当前库存量。

  4. 订单详情页的时间轴在某些订单状态下缺少关键节点。比如已取消的订单没有显示取消原因和时间。修复后完善了各个状态的时间轴节点展示。

  5. 管理员批量操作时没有进度反馈。如果选中的订单数量较多，批量确认或批量发货操作需要等待一段时间，但没有加载状态显示。修复后添加了操作按钮的加载状态。
